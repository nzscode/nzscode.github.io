    Important Info:

  • It is important to be in the database where you want to create the schema.
    A database can contain 1 or more schemas, but a aschema can belong to only one database.
  • The default schema is public.
  • To create a new schema:
    • CREATE SCHEMA <new schema name>;
  • IF you try to look up the tables using \dt, you wouldn't see the new schema. This is because the search path is set to public by default.
  • To show the current search path:
    • SHOW search_path;
  • To put the new schema in the path:
    • SET search_path TO <new schema name>;
  • Or to use multiple schemas, so that when \dt is used all tables in all schemas will show: 
    • SET search_path TO <new schema name>, public;
  • To see all Schemas use \dn

  • When creating a table, you must always specify the columns and their types.
  • To create a table:
    • CREATE TABLE IF NOT EXISTS <table name>(

  • To create a table in a schema:
    • CREATE TABLE IF NOT EXISTS <schema name.table name>(

  • Serial can have the PRIMARY KEY attribute, which will help it when joining it with other tables for queries.
       The PRIMARY KEY makes each record unique.
  • NOT NULL => this field must have data in it, if there is no data, the table will not work.
  • DEFAULT => Default value to fill as value for the column if no data is provided at input.

      • The most often used Data Types in SQL tables.
  • Serial: 1 to 2Billion, consecutive auto incrementing number.
  • Integer: From -21,000 upto 21,000.
  • Decimal: Upto 13,000 before decimal, and 16,000 after decimal.
  • Money: From -92,000 upto 92,000.
  • Varchar(n): Variable length string where (n) is the user specified max length of characters.
  • Text: unlimited length of characters.
  • Bool: True / False.
  • Date: Saved in yy-mm-dd format.
  • Time(p)[without time zone]: Time of day no time zone.
  • Time(p): Time of day, including time zone.
  • Timestamp(p)[without time zone]: Date and time, without time zone.
  • Timestamp(p): Date and time, including time zone.
  • Json: Textual json data.
  • XML: XML data.

  • To rename table
    • ALTER TABLE IF EXISTS <table name> RENAME TO <new table name>;
  • To rename table in a schema
    • ALTER TABLE IF EXISTS <schema name.table name> RENAME TO <schema name.new table name>;
  • Set Schema of a table
    • ALTER TABLE IF EXISTS <table name> SET SCHEMA <schema name>;
  • Column add, drop, and alter can also be done at the same time.
  • To add a column:
    • ALTER TABLE IF EXISTS <table name> ADD COLUMN IF NOT EXISTS <column name> <column type> ;
  • To add a column in a schema:
    • ALTER TABLE IF EXISTS <schema name.table name> ADD COLUMN IF NOT EXISTS <column name> <column type> ;
  • To drop a column:
    • ALTER TABLE IF EXISTS <table name> DROP COLUMN IF EXISTS <column name> ;
  • To drop a column in a schema:
    • ALTER TABLE IF EXISTS <schema name.table name> DROP COLUMN IF EXISTS <column name> ;
  • To alter column name:
    • ALTER TABLE IF EXISTS <table name> RENAME COLUMN <new column name> ;
  • To alter column name in a schema:
    • ALTER TABLE IF EXISTS <schema name.table name> RENAME COLUMN <new column name> ;
  • To alter column type:
    • ALTER TABLE IF EXISTS <table name> ALTER COLUMN <column name> TYPE <column type>;
  • To alter column type in a schema:
    • ALTER TABLE IF EXISTS <schema name.table name> ALTER COLUMN <column name> TYPE <column type>;
