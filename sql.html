    Using GitBash/Shell :

    Important Info:

  • It is important to be in the database where you want to create the schema.
    A database can contain 1 or more schemas, but a aschema can belong to only one database.
  • The default schema is public.
  • To create a new schema:
    • CREATE SCHEMA <new schema name>;
  • IF you try to look up the tables using \dt, you wouldn't see the new schema. This is because the search path is set to public by default.
  • To show the current search path:
    • SHOW search_path;
  • To put the new schema in the path:
    • SET search_path TO <new schema name>;
  • Or to use multiple schemas, so that when \dt is used all tables in all schemas will show: 
    • SET search_path TO <new schema name>, public;
  • To see all Schemas use \dn

  • To rename table
    • ALTER TABLE IF EXISTS <table name> RENAME TO <new table name>;

    • ALTER TABLE IF EXISTS <schema name.table name> RENAME TO <schema name.new table name>;
  • Set Schema of a table
    • ALTER TABLE IF EXISTS <table name> SET SCHEMA <schema name>;
  • To move a table
    • ALTER TABLE IF EXISTS <table name> SET SCHEMA <schema name>;
  • To move a table from one schema to another
    • ALTER TABLE IF EXISTS <schema name.table name> SET SCHEMA <schema name>;
  • Column add, drop, and alter can also be done at the same time.
  • To add a column:
    • ALTER TABLE IF EXISTS <table name> ADD COLUMN IF NOT EXISTS <column name> <column type> ;
  • To add a column in a schema:
    • ALTER TABLE IF EXISTS <schema name.table name> ADD COLUMN IF NOT EXISTS <column name> <column type> ;
  • To drop a column:
    • ALTER TABLE IF EXISTS <table name> DROP COLUMN IF EXISTS <column name> ;
  • To drop a column in a schema:
    • ALTER TABLE IF EXISTS <schema name.table name> DROP COLUMN IF EXISTS <column name> ;
  • To alter column name:
    • ALTER TABLE IF EXISTS <table name> RENAME COLUMN <new column name> ;
  • To alter column name in a schema:
    • ALTER TABLE IF EXISTS <schema name.table name> RENAME COLUMN <new column name> ;
  • To alter column type:
    • ALTER TABLE IF EXISTS <table name> ALTER COLUMN <column name> TYPE <column type>;
  • To alter column type in a schema:
    • ALTER TABLE IF EXISTS <schema name.table name> ALTER COLUMN <column name> TYPE <column type>;
  • To alter column default value:
    • ALTER TABLE IF EXISTS <table name> ALTER COLUMN <column name> SET DEFAULT <value>;
  • To alter column default value in a schema:
    • ALTER TABLE IF EXISTS <schema name.table name> ALTER COLUMN <column name> SET DEFAULT <value>;
  • To drop column default value:
    • ALTER TABLE IF EXISTS <table name> ALTER COLUMN <column name> DROP DEFAULT;
  • To drop column default value in a schema:
    • ALTER TABLE IF EXISTS <schema name.table name> ALTER COLUMN <column name> DROP DEFAULT;

  • Where clause uses comparison and logical Operators.
  • =
  • <
  • >

  • <> or !=
  • AND
    • Used when checking if all conditions are either TRUE or FALSE 
    SELECT * FROM <table name> WHERE <column name> = <value> AND <column name> = <value>;
  • OR
    • Used if at least one condition must evaluate to TRUE 
    SELECT * FROM <table name> WHERE <column name> = <value> OR <column name> = <value>;
  • IN
    • Used only when checking a list of operators. Returns TRUE if value matches any value in the list. 
    SELECT * FROM <table_name> WHERE <column name> IN (<value1, value2, ...>);
  • BETWEEN
    • Used to match a value between a range. 
    SELECT * FROM <table name> WHERE <column name> BETWEEN <value1> AND <value2>;
  • LIKE
    • Used to query data using pattern matching.
  • %
    Matches any sequence of zero or more characters.
    EX: Matches anything that starts with 'Vio' 
    SELECT * FROM <table name> WHERE <column name> LIKE 'Vio%';
  • _
    Matches any single character
    Matches anything where the second character is 'i', followed by any number of other characters. 
    SELECT * FROM <table name> WHERE <column name> LIKE '_i%';
  • ILIKE Used for non-case-sensitive matching.
    Matches but doesn't care for case sensitive. 
    SELECT * FROM <table name> WHERE <column name> ILIKE '%Vio%';

  • IS NULL
    • Check if the value is NULL. 
    SELECT * FROM <table name> WHERE <column name> = <value> AND <column name> IS NULL;
    Return results including IS NULL. 
    SELECT * FROM <table name> WHERE <column name> IS NULL OR/AND <column name> IN (<value1>, <value2>, <...>);
  • NOT
    • Omits the values specified by NOT 
    SELECT * FROM <table name> WHERE <column name> NOT IN (<value1, value2, ...>);
